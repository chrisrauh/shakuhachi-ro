Hi there! I'm Christian.

I am a shakuhachi student and web developer building this website: shakuhachi.ro.

This is a place to discover, create and share shakuhachi scores. It's inspired by [The Sessions](https://thesession.org/), Flat.io and vexflow/vextab, but for shakuhachi.

## Forking

If you've used GitHub or similar tools, forking will be familiar. If not, any score here can be forked, which gives you your own editable copy. Handy when you want to edit, transpose, or adapt a piece. Your version is independent; it's linked to the original, but you won't affect it. it's handy to create versions of the same song. 

## Embedding scores

A big part of why I built this is the score renderer — a standalone web component for displaying shakuhachi notation on any webpage. This live example shows it in action:

It's designed so anyone can embed scores in a blog post, a comment, or their own tool. More in the [GitHub project](https://github.com/chrisrauh/shakuhachi-ro).

## On AI

To be honest, this site wouldn't exist without it. I'd had the idea for years but never enough time. Claude Code let me turn the small pockets of time I have into something real.

I understand that's not neutral ground for everyone. The technology raises genuine questions and I don't dismiss them. So I want to be upfront about how this was made.
